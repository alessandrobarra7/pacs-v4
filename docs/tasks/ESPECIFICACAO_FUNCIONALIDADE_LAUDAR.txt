PARTE 2 — O QUE FALTA: GAPS ENTRE SITUAÇÃO ATUAL E OBJETIVO
============================================================

GAP 1 — POUCOS TEMPLATES (o mais impactante)
  Situação: 6 templates globais para 96 tipos de exame
  Objetivo: ~60-80 templates cobrindo os exames mais laudados
  Impacto: médico abre laudo de "RM de Joelho" e não encontra template
            → digita tudo do zero → lentidão e inconsistência

GAP 2 — SEM CAMPO CATEGORY (anatomia)
  Situação: templates agrupados apenas por modality (RX, TC, US, RM)
  Objetivo: agrupamento por região anatômica DENTRO de cada modalidade
            Ex: RM → Joelho, RM → Crânio, TC → Tórax, TC → Abdome
  Impacto: médico vê lista plana de templates sem organização

GAP 3 — SEM AUTO-SUGESTÃO DE TEMPLATE
  Situação: médico seleciona exame na aba Exame, mas a aba Conteúdo
            não filtra automaticamente os templates para aquele exame
  Objetivo: ao selecionar "RM de Joelho", a aba Conteúdo mostra
            automaticamente os templates de RM de Joelho no topo
  Impacto: médico precisa navegar manualmente para encontrar o template

GAP 4 — SEM TELA ADMINISTRATIVA DE TEMPLATES
  Situação: admin_master cria templates via editor interno à aba Conteúdo
            (mesma interface do médico, com um botão "Global")
  Objetivo: tela dedicada no painel admin para criar/organizar templates
            por categoria, vincular a unidades, visualizar com preview
  Impacto: gestão de templates está misturada com o fluxo do médico

GAP 5 — TEMPLATES SEM SEÇÕES ESTRUTURADAS
  Situação: bodyTemplate é HTML livre — médico pode escrever qualquer coisa
  Objetivo: templates com seções fixas (Técnica / Achados / Impressão)
            com placeholder de texto que o médico substitui
  Impacto: laudos inconsistentes entre médicos da mesma unidade

GAP 6 — FRASES NÃO VINCULADAS AO EXAME SELECIONADO
  Situação: FrasesTab mostra todos os grupos de frases (RX, TC, US, RM)
            sem filtrar pelo exame atual
  Objetivo: ao laudar "TC de Crânio", frases de TC aparecem primeiro
            e destacadas; frases de US ficam minimizadas
  Impacto: médico percorre lista longa para achar a frase certa

GAP 7 — DOCUMENTO DO ORIENTACAO PROPÕE COISAS INCORRETAS
  Estas partes do documento devem ser IGNORADAS:
  ✗ Tabela user_customizations — REDUNDANTE. Assinatura, carimbo
    e frases já existem nas tabelas users e phrases.
  ✗ API REST (POST /api/layouts/create) — ERRADO. O projeto usa tRPC.
    Criar REST quebraria type-safety e auth middleware.
  ✗ Adicionar content/category em model_layouts — ERRADO. model_layouts
    é layout visual (fundo, margens). Conteúdo fica em templates.
  ✗ "usuário root" — não existe. O role correto é admin_master.

PARTE 3 — LIMPEZA DE FUNCIONALIDADE (o que simplificar)
============================================================

REMOVER OU SIMPLIFICAR DO EDITOR DO MÉDICO:
────────────────────────────────────────────

  SIMPLIFICAR — Formulário de criação de template pessoal
    Situação: dentro da aba Conteúdo, o médico pode criar template
              com nome, modalidade, título e bodyTemplate completo
    Problema: criar um template inteiro dentro do editor de laudo é
              trabalhoso — o médico quer salvar o laudo atual como
              template, não criar um do zero
    Solução: substituir o formulário complexo por um único botão
             "Salvar este laudo como meu template"
             → popup simples pede apenas: nome e opcionalmente
               uma categoria/tag para o template
             → o bodyTemplate é o conteúdo atual do editor

  REMOVER — Toggle Templates/Frases interno à ConteudoTab
    Situação: ConteudoTab tem dois botões internos (Templates | Frases)
    Problema: dois níveis de navegação — aba Conteúdo + toggle interno
    Solução: exibir Templates e Frases na mesma tela da aba Conteúdo
             como seções colapsáveis, sem toggle. O template fica no
             topo (mais usado), frases logo abaixo.

  SIMPLIFICAR — Aba Imagens
    Situação: aba Imagens tem logos da unidade, carimbo e assinatura
    Solução: manter, mas mover o botão de assinatura para a área fixa
             abaixo do editor (ao lado de Salvar/Assinar), não como aba.
             Assinatura é assinada UMA VEZ no final — não durante a edição.

  MANTER COMO ESTÁ:
    ✓ Aba Exame (árvore de exames) — bem implementada
    ✓ Editor contentEditable — funciona corretamente
    ✓ Salvar rascunho / Assinar — fluxo correto
    ✓ Laudo multi-exame — funcional

PARTE 5 — FLUXO IDEAL DO MÉDICO APÓS AS MELHORIAS
============================================================

  1. Médico clica em "Laudar" no estudo
  2. Editor abre → aba Exame ativa
  3. Médico clica em "RM" → "Extremidades" → "Joelho"
     → exam_title = "Ressonância Magnética de Joelho" é definido
     → aba Conteúdo muda automaticamente e mostra templates de RM/Joelho
  4. Médico clica no template "RM Joelho — Normal"
     → bodyTemplate com Técnica, Achados e Impressão é inserido
  5. Médico edita apenas o que difere do normal (ex: "rotura parcial de menisco medial")
  6. Se precisar de uma frase: rola para baixo na aba Conteúdo → seção Frases
     → frases de RM aparecem primeiro (filtradas pela modalidade)
  7. Médico insere assinatura (botão fixo abaixo do editor)
  8. Clica "Assinar" → laudo salvo e assinado

  TEMPO ESTIMADO COM AS MELHORIAS: 2-5 minutos por laudo normal
  TEMPO ATUAL SEM TEMPLATES: 10-20 minutos (digitação do zero)
